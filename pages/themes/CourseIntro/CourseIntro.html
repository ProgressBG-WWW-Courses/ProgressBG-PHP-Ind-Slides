PHP

PHP dev environments for Windows Users

XAMPP - the "industry standard" for beginners or those who need a broad stack immediately:
One-Click Setup: It is famously easy to install and "just works."
Full Suite: If your project specifically requires an FTP server (FileZilla) or a Mail server (Mercury) locally, XAMPP includes these by default.
Consistency: If you frequently switch between Windows and macOS/Linux, using XAMPP on both provides a familiar interface.
WampServer - is the professional's choice for a few key reasons:
Version Management: It is incredibly easy to swap between different versions of PHP, MySQL, or MariaDB with a few clicks from the taskbar icon. This is vital if you maintain multiple projects that require different PHP environments.
Cleanliness: It doesn't install extra services you likely won't use (like Mercury Mail or Tomcat), keeping your system less cluttered.
Native Integration: Since it is built specifically for Windows, it tends to be more stable and performant on the OS than the cross-platform XAMPP.
Laragon - the modern WAMP
Lightweight & Fast: It is known for its speed and minimal resource usage.
Modern Features: Includes built-in tools for SSL, virtual hosts, and database management.
Portable: Can be run from a USB drive, making it great for developers on the go.

Working with Git and Github

Какво е git и как улеснява работата?
Git е инструмент, който записва всички промени по един проект. Това позволява да се връщаме назад, да работим безопасно и да споделяме работата си с други хора. Най-просто казано - Git пази "история" на проекта.
Защо да използваме Git?
История на промените: можем да видим какво е променено, кога и от кого.
Безопасни експерименти: можем да тестваме идеи, или да оправяме бъгове, без риск да счупим проекта.
Работа в екип: няколко души могат да работят по един проект, без да си пречат.

Как работи git?

Общ преглед

Working Copy:
Файловете, по които работите локално.
Repository:
Историята и файловете на проекта - локално или remote.
Local Repository:
Пълно копие на историята на проекта на вашата машина.
Remote Repository:
Онлайн/мрежово копие, използвано за споделяне.
Commit:
Записване на промени в локалното репо.
Pull:
Изтегляне на промени от remote.
Push:
Качване на локалните commit-и в remote.

Какво е GitHub?

GitHub е онлайн място, където качваме проектите си, за да ги съхраняваме, споделяме и организираме.
Може да се мисли като "Google Drive за Git проекти", улесняващ колаборацията на различни програмисти върху даден проект.

Инсталиране и настройка на Git

Инсталиране и настройка на Git

Инсталиране на Git под Windows

Изтеглете Git от официалния сайт: git-scm.com.
Процесът е стандартен - Next → Next → Finish.
Проверка на инсталацията:
git --version

Инсталиране на Git под Linux

Linux (Debian/Ubuntu):

                sudo apt-get update
                sudo apt-get install git
Справочник: Install git on Linux

Настройка на user name и email

Задаваме user name и email, за да може Git да отбелязва кой прави промените.

				git config --global user.name "Your Name"
				git config --global user.email "you@mail.com"
Проверка:

				git config user.name
				git config user.email
Добра практика при работа с Github е да използваме email, който е свързан с Github профила.

Настройка на VSCode като Git editor

За редактиране на commit съобщения във VSCode:
git config --global core.editor "code --wait"
